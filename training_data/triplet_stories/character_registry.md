# Story Character Registry

Canonical registry for recurring named characters introduced in Tier 2 and reused in Tier 3 and Tier 4.

Purpose:
- keep character naming consistent across story tiers
- prevent the same domain/story thread from silently changing names between tiers
- let Tier 2 introduce names without forcing all character decisions immediately

Use this file together with:
- `training_data/triplet_stories/tier_2/review_queue.md`
- `training_data/wiki/story_layer_rules.md`
- `training_data/phases/phase_6/story_dialogue_progression.md`

## Rule

If a Tier 2 story introduces a named character for a recurring role in a domain/story thread, the same character should be reused in the matching Tier 3 and Tier 4 versions unless a human explicitly decides otherwise.

Example policy:
- Tier 1: `a boy rides the bus`
- Tier 2: `a boy named Timmy rides the bus`
- Tier 3: keep `Timmy` for the bus story thread
- Tier 4: keep `Timmy` for the bus story thread

Do not rename recurring characters casually once a Tier 2 name is chosen.

## What to track

For each recurring character, record:
- character name
- domain file
- story/triplet ID or anchor
- role description
- first tier introduced
- notes on family/friends/places if needed for consistency
